## Classification standard

| Classification    | Standard used in this audit                                                                                                          |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| `PROVEN`          | Supported by inspected repository behavior or direct browser verification, within the scope stated.                                  |
| `MODELED`         | A synthetic example, representative count, identifier, record, or interface state; it is not customer evidence or a measured result. |
| `PLANNED`         | A product direction or design objective that is not represented as production capability.                                            |
| `POSITIONING`     | Category, audience, or differentiation language that describes intended market position rather than verified operation.              |
| `UNSAFE / REMOVE` | Unsupported, operationally unverified, privacy-sensitive, patent-sensitive, or likely to be mistaken for production fact.            |

Risk labels refer to publication risk: `Low` means the claim is appropriately supported and scoped; `Moderate` means qualification and context must remain adjacent; `High` means the statement must not be strengthened without evidence; `P0` means the unresolved item blocks publication.
